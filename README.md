# PlanForge

**Turn a one-line idea into a build-ready product spec — on your own machine.**

PlanForge is a local-first desktop app that expands a single sentence ("a membership SaaS for
neighborhood gyms") into a complete, developer-ready planning document: service overview, user &
admin flows, DB schema, security, privacy-law review, REST API spec, architecture (Mermaid),
CRUD mapping, NFR/SLO targets, a testing strategy, deployment & observability, and a
risks/roadmap section — generated by an LLM behind a senior-engineer system prompt.

## Why PlanForge

- **One line in, a full spec out.** Thirteen structured sections, not a vague outline — each section is an
  *execution spec* ("who / what / how"), with async-by-default heavy work, error contracts, and a
  Korean-privacy-law review baked into the system prompt.
- **Local-first.** Your idea, the generated docs, and your API key never leave your machine. The
  backend runs inside the app.
- **Bring your own brain.** Use a **local model via Ollama** (no key, no cost) or plug in an
  **Anthropic API key** for higher quality — switch anytime in Settings.
- **Bilingual.** Full Korean / English UI toggle.

## Install

Grab the latest installer from the [**Releases**](../../releases) page:

| OS | File |
|----|------|
| macOS (Apple Silicon / Intel) | `PlanForge_*.dmg` |
| Windows | `PlanForge_*-setup.exe` |

> **Unsigned build note:** until the app is code-signed, macOS shows "unidentified developer"
> (right-click → **Open** to bypass) and Windows shows a SmartScreen prompt (**More info → Run anyway**).

### Pick your AI engine (first run)

PlanForge needs a language model. Two options — choose in **⚙ Settings**:

1. **Ollama (local, free, no key)** — install [Ollama](https://ollama.com), then:
   ```bash
   ollama pull llama3.1
   ```
   PlanForge talks to it at `localhost:11434`. This is the default.
2. **Anthropic (cloud, higher quality)** — paste your API key in Settings. It's stored only in
   `~/.planforge/config.json` and never transmitted anywhere except Anthropic.

> Why a model is required: a cloud LLM (Claude) can't be bundled into an app, and a local LLM runs
> through Ollama. PlanForge bundles the *server*, not the model.

## How it works

```
┌──────────────┐     HTTP      ┌───────────────────────┐
│  Tauri shell │ ───────────▶ │  FastAPI sidecar       │
│  (Next.js UI)│  127.0.0.1   │  (bundled, local)      │
└──────────────┘   :8000      │   • async job pipeline │
        ▲                      │   • SQLite (~/.planforge) │
        │ progress (SSE)       │   • LLM client ────────┼──▶ Ollama (:11434)
        └──────────────────────┤                        │    or Anthropic API
                               └───────────────────────┘
```

- The desktop shell ([Tauri](https://tauri.app)) spawns a **PyInstaller-bundled FastAPI backend** as a
  child process on launch and stops it on exit.
- Generation is **asynchronous**: the UI gets a `job_id`, streams progress over SSE, then renders the
  parsed sections.
- Engine selection is swappable at runtime (Ollama / Anthropic / stub).

## Features

- Async generation pipeline with live progress (SSE)
- Per-section **refine** (rewrite one section from a follow-up request)
- Markdown / JSON **export** of the assembled document
- Admin view (user approval, job failure-rate, prompt versions)
- Usage accounting + per-user rate limiting
- Account deletion with privacy-law-aware retention
- KO / EN i18n (UI + localized API errors)

## Run from source

Prereqs: **Node 18+**, **Rust** ([Tauri prerequisites](https://tauri.app/start/prerequisites/)), **Python 3.12**.

```bash
# 1) Backend deps. In dev, Tauri runs the backend with `python -m uvicorn`, so a
#    `python` on PATH that can import the backend is required. A venv is the easiest
#    way — it also provides a `python` shim (some installs only ship `python3`).
cd backend
python3 -m venv .venv
source .venv/bin/activate            # Windows: .venv\Scripts\activate
pip install -r requirements-dev.txt
cd ..

# 2) Run. The first `npm run tauri:dev` auto-generates the app icons and a
#    placeholder sidecar (both gitignored), so there's no manual `tauri icon` or
#    PyInstaller step. Keep the venv activated in this shell so `python` resolves.
cd desktop-app
npm install
npm run tauri:dev
```

Build local installers and CI release details: see **[desktop-app/README.md](desktop-app/README.md)**.
Backend API details: see **[backend/README.md](backend/README.md)**.

## Privacy

- Everything runs locally. Your idea and generated documents are stored in a local SQLite file under
  `~/.planforge`.
- API keys are stored locally and are sent only to the provider you choose.
- Account deletion purges personal data immediately; only pseudonymised operational logs are retained.
